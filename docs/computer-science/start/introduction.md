# 计算机基础 · 介绍

> 从计算机最基础的内容出发，掌握代码开发的内功

## 一句话概况

计算机基础不是为了应付面试，而是帮你看懂"为什么浏览器这样设计、为什么网络这样工作"——有了这层理解，你才能从代码执行者变成系统设计者。

## 这一章讲什么

前端开发者学计算机基础，有一个很常见的误区：要么完全不学，觉得"我又不做后端"；要么照着 408 教材从头啃，啃完发现和工作没什么关系。

这一章的目标是**以前端视角为入口**，选择那些和日常开发强相关的基础内容：HTTP 为什么会有缓存问题、TCP 三次握手为什么影响首屏加载、设计模式在 React 源码和 Webpack 插件系统里怎么体现、HTTPS 证书为什么浏览器会报红……这些才是真正让人"豁然开朗"的知识。

数据结构与算法单独拆出去了（见算法模块），这里不重复。浏览器渲染原理、JS 事件循环这类内容放在前端基础里面——它们属于"JS 运行环境"，不是通用计算机基础。这一章专注于：**网络协议、安全机制、设计模式、操作系统轻量基础、编译原理基础（AST 层面）**。

这五块内容在工程里彼此关联：理解了 TCP 才能真正看懂 HTTP/2 多路复用；理解了 TLS 握手才能明白为什么 HTTPS 首字节时间比 HTTP 长；理解了 AST 才能看懂 Babel 插件和 ESLint 规则是怎么运作的。

## 学习建议

- **先学 HTTP，再延伸 TCP**。绝大多数前端开发者日常遇到的问题都在 HTTP 层：缓存、跨域、状态码。TCP 只需要理解到"三次握手影响连接建立耗时、HTTP/2 为什么能多路复用"就足够了，不需要去背滑动窗口算法。

- **设计模式不要从《GoF 23种》开始**。先从"你在 React 里已经用过的模式"出手——Context API 是观察者模式、HOC 是装饰器模式、Hooks 可以理解为策略模式的一种实现——有了代码锚点，再回头看模式的定义会清晰很多。

- **安全这块要动手复现，不要只看定义**。XSS 和 CSRF 看完之后，在本地用一个简单的 Express + HTML 页面手动跑一次攻击过程，你会真正理解为什么 `HttpOnly`、`SameSite`、CSP 头各自能防什么，光看概念很难建立直觉。

- **编译原理先看 AST，不要从词法文法开始**。推荐用 [astexplorer.net](https://astexplorer.net) 把一段 JS 代码粘进去看树状结构，再手写一个最简单的 Babel 插件，比看理论书效率高得多。这块内容知道到"能读懂 Babel / ESLint 插件源码"就够用，不需要去实现语法解析器。

- **操作系统部分够用即可**。前端最相关的只有两块：JS 单线程和浏览器多线程（Web Worker）的关系、Node.js 的 I/O 模型（事件驱动非阻塞）——搞清楚这两块就行，进程调度算法不是优先项。
